# 第二周作业 LoginProject
## 1. 项目概述
本项目是一个功能完备的Android登录/注册模块，旨在模拟真实应用的用户认证流程，涵盖了从UI设计、用户交互到后端数据持久化的完整开发链路。

## 2. UI层实现：
- 布局：采用 ConstraintLayout 进行复杂的响应式布局设计，确保在不同屏幕尺寸和密度下都能保持一致的用户体验。
- 组件：使用了Google Material Design Components 以打造现代化且符合设计规范的UI。
- 资源管理：在适宜的地方使用了 VectorDrawable图标替代传统位图，有效减小APK体积了并保证图标在不变通分辨率下的清晰度。在 Drawable 里使用相关技术创建了自定义的UI效果（如圆角背景、点击波纹等等）。

## 3. 功能模块与技术实现
- 用户注册模块：
  - UI：通过TextInputEditText和TextInputLayout实现了带有输入提示、错误校验和密码可见性切换功能的表单。
  - 逻辑：对用户名、邮箱、密码格式进行前端校验，确保数据合法性。
  - 数据存储：将注册成功的用户信息通过ContentValues封装，存储到本地SQLite数据库中。
- 用户登录模块：
  - UI：写了具有可复用性的UI组件，以减少项目复杂度和提供流畅的输入体验。
  - 逻辑：从数据库中查询用户信息，通过对比输入的用户名和密码，完成身份验证。
  - 状态管理：登录成功后，通过SharedPreferences 持久化存储用户登录状态（如用户ID或Token），实现了自动登录功能，避免用户每次打开应用都需重新登录。
- 个人中心/主页模块：
  - UI：展示用户的基本信息（如用户名、头像），并提供“退出登录”功能。
  - 数据交互：从SharedPreferences 中读取用户ID，再从SQLite数据库中查询并展示对应的用户详细信息。
  - 导航：实现“退出登录”逻辑，清除SharedPreferences中的登录状态，并导航回登录界面。

## 4. 数据持久化
- SQLite数据库：
  - 角色：作为应用的核心数据存储引擎，用于持久化存储所有用户信息。
  - 实现：通过继承SQLiteOpenHelper类来管理数据库的创建、版本升级。自定义了DatabaseHelper类，封装了对用户表的增（addUser）、查（checkUser）等原子操作，实现了数据访问层的基本抽象。
- SharedPreferences：
  - 角色：作为轻量级的键值对存储方案，专门用于存储用户的登录凭证和会话状态。
  - 实现：通过简单的API调用，高效地读写用户的登录状态。

## 5. 项目亮点与可扩展性
- 现代UI/UX实践：项目遵循Material Design设计准则，实现了干净、直观的用户界面和流畅的交互体验。
- 代码规范与封装：通过创建DatabaseHelper类将数据库操作与业务逻辑分离，体现了良好的代码分层思想。
- 可扩展性：采用了单Activity架构，为未来扩展更复杂的业务功能（如集成网络请求、引入依赖注入框架等）打下了坚实的基础。
